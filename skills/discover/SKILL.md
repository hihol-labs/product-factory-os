# Discover

Create `DISCOVERY.md` for a product idea.

## Process

1. Clarify target user, painful problem, current alternatives, budget, timeline, and success metric.
2. Identify user segments and primary persona.
3. Summarize alternatives and differentiation.
4. Prioritize features with MoSCoW and a lightweight RICE score.
5. Define the MVP scope, non-goals, and kill criteria.

## Output

Write or update `DISCOVERY.md` with:

- Problem statement
- Target users
- Jobs to be done
- Alternatives and competitors
- Positioning
- MVP scope
- Feature priority table
- Risks and unknowns

## Rules

- If facts are unknown, mark them as assumptions.
- Do not invent market numbers without research.
- Ask before overwriting an existing `DISCOVERY.md`.
